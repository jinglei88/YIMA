# 易码架构设计（当前实现）

## 1. 解释器流水线

1. `词法分析.py`：把源码切成 Token。
2. `语法分析.py`：把 Token 解析成 AST。
3. `解释器.py`：遍历 AST 并执行。

入口：`易码.py`

## 2. 目录结构

```text
yima/
├── 词法分析.py
├── 语法分析.py
├── 语法树.py
├── 解释器.py
├── 环境.py
├── 错误.py
└── 信号.py
```

## 3. 词法层（Lexer）

职责：

1. 识别关键字、标识符、数字、字符串、符号。
2. 识别缩进层级并产生 `缩进/退缩` Token。
3. 支持 `#` 与 `标注：` 注释。
4. 支持模板字符串占位符（`【变量】`）。

注意：

- 逻辑词 `并且/而且/或者/取反` 是语法入口。
- 算术/比较目前以符号入口为主（`+ - * / > < ==` 等）。

## 4. 语法层（Parser）

职责：

1. 构建语句 AST：赋值、条件、循环、函数、异常、模块、图纸。
2. 构建表达式 AST：调用、属性访问、索引访问、运算。
3. 处理优先级：算术 > 比较 > 逻辑。

关键语法点：

- 遍历语法固定为 `遍历 ... 里的每一个 叫做 ...`
- 代码块由缩进 Token 驱动
- `返回` 在函数体内通过控制流信号回传值

## 5. 运行时（Interpreter）

职责：

1. 在 `环境.py` 作用域链上读写变量。
2. 执行内置函数、用户函数、图纸实例方法。
3. 处理控制流信号：`停下信号` `略过信号` `返回信号`。
4. 支持模块导入、GUI 内置、画图库内置。

模块导入行为：

- `引入 "x"`：先查内置虚拟模块，再查 `.ym` 文件，再查 Python 模块。
- 当前内置虚拟模块：`文件管理`、`系统工具`、`数据工具`、`网络请求`、`本地数据库`、`图形界面`、`画板`、`魔法生态库`。
- 其中：`网络请求` 提供文本与 JSON 快捷请求；`本地数据库` 支持自动提交与显式事务（开始/提交/回滚）。
- `.ym` 搜索路径：当前工作目录、`示例/`。
- `.ym` 模块导入后以命名空间字典注册，可用点语法访问。

## 6. 约束与边界

1. 当前版本不维护历史同义词兼容层。
2. 文档示例必须以“可直接运行”为标准。
3. 语法改动必须同步更新：`词法分析.py`、`语法分析.py`、`文档/语言规范.md`、`示例/`。
