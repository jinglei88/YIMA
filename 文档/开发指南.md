# 易码开发指南（用户版，真实实现）

本指南面向“要用易码开发真实软件”的用户，不讲编译器内部，只讲可直接落地的做法。

## 1. 你现在可以做什么

当前实现已经可覆盖以下软件类型：

1. 命令行工具：文本处理、数据转换、批量脚本。
2. 本地数据应用：文件读写、JSON、SQLite（事务）。
3. 联网工具：GET/POST、JSON API 对接。
4. 图形化小工具：窗口、输入框、按钮、网格表格。
5. 画板/小游戏：基于 turtle 的画图、键盘监听与交互。

## 1.1 v1.0 稳定边界（先看）

1. 核心语法已冻结：`如果/否则如果/不然`、`功能`、`定义图纸`、`引入 ... 叫做 ...`。
2. 内置能力保持兼容：已有函数名不删除、不重命名。
3. 新增能力走增量：只新增，不改旧行为语义。
4. 历史同义词不再新增，文档默认只展示官方写法。

## 2. 环境与启动

要求：

1. Python 3.9+
2. 源码 UTF-8 编码

启动：

```bash
python 易码.py
python 易码.py 示例/欢迎.ym
python 易码编辑器.py
```

## 3. 推荐项目结构

建议一开始就按“分层”来写，后续维护更稳：

```text
你的项目/
├── 主程序.ym
├── 界面/
│   └── 界面层.ym
├── 业务/
│   └── 业务层.ym
├── 数据/
│   └── 数据层.ym
└── 资源/
    └── 图标.ico
```

典型入口：

```ym
引入 "界面/界面层.ym" 叫做 界面
界面.运行()
```

## 4. 模块导入规则（真实行为）

### 4.1 统一导入

1. 官方写法：`引入 "模块名" 叫做 名称`
2. 若模块里存在同名导出（或 Python 模块同名属性），会直接绑定该名称；否则绑定整个模块对象。

### 4.2 模块查找顺序

运行时导入顺序：

1. 内置虚拟模块（如 `图形界面`、`本地数据库`）。
2. 本地 `.ym` 模块（当前工作目录及 `示例/`）。
3. Python 模块（如 `math`、`time`）。

### 4.3 导入建议

1. 本地模块建议写相对名：`引入 "数据/数据层.ym" 叫做 数据`
2. 需要直接拿单个能力时，把名称写到 `叫做` 后面（如 `引入 "math" 叫做 pi`）。
3. 若提示未定义，优先检查：模块名、别名拼写、全角/半角。

## 5. 开发软件的推荐流程

1. 先做数据层：先把文件/数据库读写跑通。
2. 再做业务层：把规则封装成 `功能`。
3. 最后接界面层：按钮只负责调用业务功能。
4. 每完成一个功能就运行，别堆大改动再调试。
5. 交付前跑自动回归，最后再走一遍手工交互。

### 5.1 什么时候需要改解释器

只有这类需求才需要改 `yima/解释器.py`：

1. 新语法关键字（例如新增一种循环语句）。
2. 新内置函数（例如新增系统能力入口）。
3. 解释器级 bug 修复（语言行为本身错误）。

下面这些都不需要改解释器：

1. 业务软件逻辑（应写在 `.ym` 项目文件）。
2. 项目模块拆分（`引入`）。
3. 使用 Python 生态库（通过 `引入 "模块名" 叫做 别名`）。

### 5.2 打包原则（避免“写完还要改内核”）

当前打包器会自动做两件事：

1. 递归打包项目目录下的 `.ym` 文件（保留目录结构）。
2. 扫描 `.ym` 中 `引入` 语句，自动追加 Python `hidden-import`。

建议：把一个软件的源码放在独立项目目录内，再从该目录主入口打包。

### 5.3 全局变量与局部变量（开发者必读）

易码当前没有单独的 `global/local` 声明关键字，作用域由“写在什么位置 + 运行时开关”决定。

1. 顶层赋值：全局变量（整个脚本可见）
2. `功能` 的参数：局部变量
3. `功能` 内新建变量：局部变量
4. `功能` 内给同名变量赋值：默认会向上覆盖外层同名变量（兼容历史行为）

示例 A：默认模式（会改到外层同名变量）

```ym
总分 = 0

功能 加分()
    总分 = 总分 + 10

加分()
显示 总分   # 默认输出：10
```

示例 B：严格局部模式（不改外层，只改当前功能内部）

```ym
总分 = 0

功能 加分()
    总分 = 总分 + 10
    显示 "功能内总分: " + 转文字(总分)

加分()
显示 "外层总分: " + 转文字(总分)   # 严格模式下外层仍是 0
```

开启严格局部作用域（Windows PowerShell）：

```powershell
$env:YIMA_STRICT_SCOPE="1"
python 易码.py 你的脚本.ym
Remove-Item Env:YIMA_STRICT_SCOPE
```

推荐实践：

1. 小脚本/教学示例：可用默认模式，写法更直观。
2. 中大型项目：建议开启 `YIMA_STRICT_SCOPE=1`，避免跨模块变量污染。
3. 共享状态尽量放进 `字典` 或 `图纸实例`，减少大量裸全局变量。
4. 需要修改共享状态时，优先通过“功能参数 + 返回值”传递，不直接依赖隐式外层覆盖。

## 6. 调试与定位

编辑器里重点看两个位置：

1. 右下角语法状态：语法是否通过。
2. 左侧“问题列表”：
   - `[错]` 语法/运行前错误定位
   - `[提]` 一般语义提示

调试控制台用于看：

1. 运行日志
2. 错误行号
3. 打包过程输出

### 6.1 编辑器补全（最新）

1. 自动联想：输入标识符会自动弹出候选，不必每次按 `Ctrl+Space`。
2. 语义排序：当前作用域内的参数/局部变量会优先出现在前面。
3. 签名提示：功能、导入成员等可调用项会显示参数签名（如 `(a, b)`）。
4. 成员补全：输入 `别名.` 会弹出该模块导出成员（例如 `系统.`、`文件.`、`数据.`、`工具.`）。
5. 输入法兼容：中文输入法下输入 `。/．/｡` 会在代码区自动转成英文 `.`，字符串和注释内不转换。
6. 键盘操作：候选列表支持 `↑/↓` 选择，`Enter` 确认，`Esc` 关闭。
7. 鼠标操作：单击候选即可插入。
8. 自动补括号：当候选是可调用能力（如功能/成员功能）时，确认后会自动补 `()` 并把光标放到括号中间。
9. 标签语义：候选前缀会区分类型（如 `成员-功能`、`导出-变量`），颜色也会按类型区分。

## 7. 打包 EXE（编辑器）

推荐步骤：

1. 先保存全部文件。
2. 在编辑器点击“导出软件(exe)”。
3. 选择导出模式：
   - 一键打包：零配置快速出包（默认输出到项目 `易码_成品软件/`）。
   - 高级导出：可自定义软件名称、输出路径、图标、运行模式。
4. 确认摘要后开始打包。

常见失败点：

1. 模块文件不在项目目录或导入路径里。
2. 第三方依赖未自动识别（用 `易码打包清单.json` 补充）。
3. 资源文件未随项目保存。

### 7.1 打包清单（可选，推荐）

当自动依赖识别不够时，可在项目根目录放 `易码打包清单.json`（或 `yima_pack.json`）：

```json
{
  "hidden_imports": ["requests", "PIL.Image"],
  "datas": [
    "assets/logo.png",
    { "src": "assets", "dest": "assets" }
  ]
}
```

说明：

1. `hidden_imports`：手动补充 Python 依赖。
2. `datas`：附加资源文件/目录到 EXE。
3. `src` 支持相对项目根目录路径。
4. 可直接参考模板：`示例/易码打包清单_模板.json`。

## 8. 自动回归与手动回归

自动回归（推荐每次发版前执行）：

```bash
python 易码.py 示例/M16全量能力测试项目/主程序.ym
```

手动交互回归（覆盖阻塞命令）：

```bash
python 易码.py 示例/M16全量能力测试项目/07_GUI手动交互验证.ym
python 易码.py 示例/M16全量能力测试项目/08_画板手动交互验证.ym
```

发布前一键回归（基础回归 + 文档一致性 + M16 + 打包冒烟）：

```bash
python scripts/run_release_regression.py
```

仅验证 v1 冻结契约（版本号/关键词/文档声明）：

```bash
python scripts/run_v1_contract_regression.py
```

仅跑文档一致性（函数数量与规范文本一致）：

```bash
python scripts/run_docs_regression.py
```

若本次不需要跑打包冒烟，可用：

```bash
python scripts/run_release_regression.py --skip-pack
```

## 9. 高概率踩坑清单

1. `绑定按键` 第一个参数要传“功能对象”，不要传字符串。
2. `打开界面` 和 `定格` 会阻塞；自动测试不要放这两个命令。
3. `弹窗输入` 返回空字符串时要做兜底处理。
4. 数据库事务要配对：`开始事务 -> 提交事务/回滚事务`。
5. 语义提示不阻止运行，但建议修掉再发布。
