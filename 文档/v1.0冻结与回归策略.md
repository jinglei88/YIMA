# 易码 v1.0 冻结与回归策略

本文档定义易码 `v1.0` 的稳定边界，目标是让开发者写出的项目在后续版本中尽量“不改代码也能跑”。

## 1. 冻结范围（v1.0）

以下内容视为 `v1.0` 核心契约，不做破坏性变更：

1. 核心关键字与句式：
`显示` `输入` `功能` `返回` `引入` `叫做`
`如果` `否则如果` `不然`
`当 ... 的时候` `重复 ... 次` `遍历 ... 里的每一个 叫做 ...`
`尝试` `如果出错`
`定义图纸` `造一个` `它的`
2. 保留字面量：
`对` `错` `空`
3. 官方导入写法：
`引入 "模块" 叫做 名称`
4. 内置函数清单（当前 126 个）的名称不做重命名、不做删除。

## 2. 兼容策略

1. 新能力只做“增量增加”，不改旧能力语义。
2. 若必须修复历史行为：
先提供兼容开关，再在文档中给迁移路径。
3. 历史同义词不再新增。
旧项目若仍使用历史写法，建议迁移到官方写法。

## 3. 版本规则

1. 主版本（MAJOR）：破坏性变更（v1.x -> v2.0）。
2. 次版本（MINOR）：新增能力且保持兼容（v1.0 -> v1.1）。
3. 修订版本（PATCH）：仅修 bug（v1.0.0 -> v1.0.1）。

当前版本文件：`VERSION`

## 4. 发布前回归（必须）

发布前至少执行一次：

```bash
python scripts/run_release_regression.py --skip-pack
```

若要包含打包冒烟：

```bash
python scripts/run_release_regression.py
```

## 5. v1 契约自动检查

新增脚本：`scripts/run_v1_contract_regression.py`

该脚本会检查：

1. `VERSION` 是否符合 `1.x.x`
2. 词法关键字契约是否完整
3. 是否未回流历史别名（如 `排列`、`是一份清单`）
4. 规范文档是否仍声明“历史同义词不再维护”
