# 易码开发者教程（完整）

本教程面向要参与易码语言开发的贡献者，内容按“能落地、能验证、能回归”编排。

## 1. 你会学到什么

1. 易码从源码到执行的完整流水线。
2. 如何新增/修改关键字或语法。
3. 如何新增内置函数。
4. 如何保证文档和实现一致。
5. 如何做最小回归，避免破坏现有示例。

## 2. 环境准备

### 2.1 运行环境

- Python 3.9+
- UTF-8 编码终端

### 2.2 启动方式

```bash
python 易码.py
python 易码.py 示例/欢迎.ym
python 易码编辑器.py
```

## 3. 项目结构与职责

```text
易码.py                # CLI 入口
易码编辑器.py          # 图形化编辑器
yima/
├── 词法分析.py         # 源码 -> Token
├── 语法分析.py         # Token -> AST
├── 语法树.py           # AST 节点定义
├── 解释器.py           # AST 执行
├── 环境.py             # 作用域/符号表
├── 信号.py             # 停下/略过/返回 控制流信号
└── 错误.py             # 词法/语法/运行时错误封装
文档/
├── 语言规范.md
├── 架构设计.md
└── 开发指南.md
```

## 4. 执行流水线（必须理解）

1. `词法分析器.分析()` 读取源码，生成 Token 序列。
2. `语法分析器.解析()` 读取 Token，生成 AST。
3. `解释器.执行()` 深度遍历 AST 并求值。
4. `环境` 维护变量查找链；`信号` 负责循环/返回跳转。

只要理解这 4 步，定位 90% 语法问题都够用。

## 5. 关键文件怎么改

### 5.1 改关键字或语法入口

先后顺序固定：

1. `yima/词法分析.py`
   - `Token类型` 新增或调整关键字枚举。
   - `多字词典` 增加关键字映射。
2. `yima/语法分析.py`
   - 在 `_解析语句` 或 `_解析基本表达式` 增加分支。
3. `yima/语法树.py`
   - 新语法需要新节点时补 AST 节点。
4. `yima/解释器.py`
   - 新增 `_做_节点名` 执行方法。

### 5.2 改内置函数

只改 `yima/解释器.py` 的 `_植入内置函数`。

- 用 `self.全局环境.记住("函数名", python_callable)` 注册。
- 注册后即可在易码脚本中直接调用。

示例（概念）：

```python
def 平方(x):
    return x * x
self.全局环境.记住("平方", 平方)
```

### 5.3 改模块导入行为

重点看：

- `_做_引入语句节点`
- `_做_精确引入语句节点`

当前行为：

1. 优先处理内置虚拟模块。
2. 其次加载 `.ym` 文件（当前目录、`示例/`）。
3. 最后尝试 Python 模块。

## 6. 新增语法实战模板

下面这套模板适用于新增一个关键字或语法形态。

### 第一步：定义词法入口

在 `Token类型` 与 `多字词典` 增加条目。

### 第二步：加入语法分支

在 `语法分析.py` 中：

- 语句级语法：接入 `_解析语句`。
- 表达式级语法：接入 `_解析基本表达式`。

### 第三步：定义或复用 AST

- 简单语法可复用现有节点。
- 语义不同就新增节点。

### 第四步：实现运行时行为

在 `解释器.py` 实现对应 `_做_...` 方法。

### 第五步：更新文档与示例

至少同步 3 处：

- `文档/语言规范.md`
- `README.md`（如面向用户）
- `示例/` 至少一个可跑样例

## 7. 常见坑与排查

### 7.1 词法识别不到关键字

排查顺序：

1. 是否已加入 `多字词典`。
2. 是否被更长词覆盖（词边界问题）。
3. 是否写错编码或全角/半角符号。

### 7.2 语法报错“读取到【就】”这类异常

通常是条件表达式没有被正确解析，优先检查：

1. 运算符是否属于当前语法入口。
2. `_解析表达式` 和优先级表是否覆盖该 token。

### 7.3 运行时“未定义名称”

优先检查：

1. 是否真的赋值到当前作用域。
2. 是否拼写一致（中文同形字、全半角）。
3. 是否在函数/图纸内部错误引用了外层变量。

## 8. 回归策略（提交前必做）

```bash
python -m py_compile 易码.py yima\词法分析.py yima\语法分析.py yima\语法树.py yima\解释器.py yima\信号.py 易码编辑器.py

python 易码.py 示例\M11模块化测试.ym
python 易码.py 示例\M12容错测试.ym
```

如果改了 GUI 或画图库，再补跑相关示例。

## 9. 文档真实性约束

文档必须满足：

1. 每个关键字都能在规范中找到“作用+写法+示例”。
2. 文档里的示例可以直接运行。
3. 文档描述与实现冲突时，以实现为准并立刻修文档。

## 10. 推荐开发流程

1. 先写要改的语法目标和边界（1 段文字即可）。
2. 改词法。
3. 改语法。
4. 改解释器。
5. 补示例。
6. 跑回归。
7. 最后修文档。

按这个顺序做，返工成本最低。
