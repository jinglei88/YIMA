# M16 全量能力测试项目

## 1. 运行方式

自动回归（推荐先跑）：

```bash
python 易码.py 示例/M16全量能力测试项目/主程序.ym
```

手动交互（覆盖阻塞命令）：

```bash
python 易码.py 示例/M16全量能力测试项目/07_GUI手动交互验证.ym
python 易码.py 示例/M16全量能力测试项目/08_画板手动交互验证.ym
```

## 2. 文件结构

- `主程序.ym`：串联全部自动测试套件并给出总统计
- `测试工具.ym`：断言、异常登记、汇总
- `模块_数学工具.ym`：本地模块导入测试依赖
- `01_核心语法与控制流.ym`：关键词与语法主干
- `02_内置函数与容器文本.ym`：转换、随机、字符串、容器函数
- `03_文件系统与JSON.ym`：文件/目录/JSON 全链路
- `04_网络与数据库.ym`：网络请求与 SQLite 事务
- `05_模块导入与图纸对象.ym`：引入/精确引入/图纸对象
- `06_GUI与画板自动检查.ym`：GUI/画板非阻塞检查
- `07_GUI手动交互验证.ym`：输入、弹窗、弹窗输入、打开界面
- `08_画板手动交互验证.ym`：画板交互与定格（方向键/重画/清屏）

## 3. 设计说明

1. 自动回归尽量覆盖全部实现命令，且保证命令行可连续执行。
2. `输入/弹窗/弹窗输入/打开界面/定格` 这类阻塞命令放在手动交互套件，避免自动回归卡住。
3. 如果当前运行环境不支持 GUI/画板，自动套件会给出跳过提示，而不是直接阻断全部测试。
4. 运行 `08_画板手动交互验证.ym` 时，请先点击画板窗口一次再按键。
